## 1. Project Overview

ShreeChoice Playworld is a full-featured, modern e-commerce application built for a toy store. It provides a seamless shopping experience for customers and a powerful dashboard for administrators to manage the store's inventory, orders, and branding.

---

## 2. Key Features

-   **Customer-Facing:**
    -   Dynamic Homepage, Advanced Shop Page with filtering, Product Quick-View, Wishlist, and a full Cart/Checkout flow.
    -   Secure user authentication (Login/Register) and Customer Profiles with order history and address management.
    -   Fully responsive design with skeleton loaders for a smooth user experience.

-   **Admin Panel:**
    -   Secure, dedicated login for administrators.
    -   Dashboard with sales and order statistics.
    -   Full management (CRUD) for Products, Brands, and Orders.
    -   Image upload functionality directly to Supabase Storage.
    -   Ability to manage "Featured" products shown on the homepage.

---

## 3. Tech Stack

-   **Frontend:** React, TypeScript
-   **Styling:** Tailwind CSS
-   **Animations:** Framer Motion
-   **Backend (BaaS):** Supabase (PostgreSQL Database, Auth, Storage)

---

## 4. Supabase Setup Instructions

To get the application's backend running, you need to set up a Supabase project. The following steps will guide you through creating the database schema and storage.

**Where to Run the Code:**
1.  Go to your project on [supabase.com](https://supabase.com).
2.  In the left-hand menu, find the icon that looks like a database cylinder and click on **SQL Editor**.
3.  Click on **+ New query**.
4.  For each step below, copy the entire SQL code block, paste it into the query window, and click the green **RUN** button.

---

**IMPORTANT NOTE:** If you are seeing errors like `Could not find the table 'public.categories'` or `could not find column 'tracking_number'`, it means your database schema is out of date. **You must run the full SQL script from Step 1 below** to update your database with the latest tables and columns required for the application to work correctly.

### Step 2: Create Storage Bucket for Images

This step is **required** for the image upload functionality in the admin panel. It creates a "bucket" (like a folder) to store product images and brand logos, and sets the permissions for who can see and upload files.

-   **Public Read Access:** Allows anyone to *view* the images, so they can be displayed on your website.
-   **Authenticated User Access:** Allows only *logged-in users* (i.e., your admin account) to upload, update, and delete images, which keeps your storage secure.

**Instructions:** Copy the entire block of code below and run it once in the SQL Editor.

```sql
-- Create a bucket named "product-images" and make it public.
INSERT INTO storage.buckets (id, name, public)
VALUES ('product-images', 'product-images', true);

-- Create a policy that allows public read access to all files in the bucket.
CREATE POLICY "Public Read Access"
ON storage.objects FOR SELECT
USING ( bucket_id = 'product-images' );

-- Create a policy that allows authenticated (logged-in) users to perform
-- ALL actions (insert, update, delete) on files in the bucket.
CREATE POLICY "Authenticated User Access"
ON storage.objects FOR ALL
TO authenticated
WITH CHECK ( bucket_id = 'product-images' );
```

### Step 3: Secure Tables with Row Level Security (RLS)

This is a **critical security step**. These rules ensure that normal users can only access their *own* data. For example, a user can only view their own orders and shipping addresses, not anyone else's.

**Instructions:** Copy the entire block of code below and run it once in the SQL Editor.

```sql
-- Enable Row Level Security on the tables that store user-specific data.
ALTER TABLE public.addresses ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.orders ENABLE ROW LEVEL SECURITY;

-- Create policies for the ADDRESSES table
-- This policy allows users to view, insert, update, and delete ONLY THEIR OWN addresses.
CREATE POLICY "Users can manage their own addresses"
ON public.addresses FOR ALL
USING (auth.uid() = user_id)
WITH CHECK (auth.uid() = user_id);

-- Create policies for the ORDERS table
-- Policy for viewing orders: Users can only see their own orders.
CREATE POLICY "Users can view their own orders"
ON public.orders FOR SELECT
USING (auth.uid() = user_id);

-- Policy for creating orders: Users can only create orders for themselves.
CREATE POLICY "Users can insert their own orders"
ON public.orders FOR INSERT
WITH CHECK (auth.uid() = user_id);
```

### Step 4: Enable Admin Access (CRITICAL)

**IMPORTANT:** You must run this query to allow the Admin account (`admin@shreechoice.com`) to view and modify ALL orders. Without this, the dashboard will not be able to update order statuses.

**Instructions:** Copy the code below and run it in the SQL Editor.

```sql
-- Allow the specific admin email to VIEW all orders
CREATE POLICY "Admin can view all orders"
ON public.orders
FOR SELECT
USING (auth.jwt() ->> 'email' = 'admin@shreechoice.com');

-- Allow the specific admin email to UPDATE all orders (e.g. changing status)
CREATE POLICY "Admin can update all orders"
ON public.orders
FOR UPDATE
USING (auth.jwt() ->> 'email' = 'admin@shreechoice.com');

-- Allow the specific admin email to DELETE all orders
CREATE POLICY "Admin can delete all orders"
ON public.orders
FOR DELETE
USING (auth.jwt() ->> 'email' = 'admin@shreechoice.com');
```

## 6. Admin Access

To access the admin dashboard, you must use a specific admin account.

1.  **Register a New User:** Go to the website's registration page and create a new user with the email `admin@shreechoice.com`. Use any password you like.
2.  **Navigate to Admin Login:** Find the hidden admin login link in the website footer (it's a small lock icon).
3.  **Log In:** Use the `admin@shreechoice.com` credentials to access the dashboard.

The `AdminLogin.tsx` component is hardcoded to only allow this specific email to access the admin portal. For a real-world application, this would be managed with user roles in Supabase.
